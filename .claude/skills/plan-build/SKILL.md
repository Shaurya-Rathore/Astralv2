---
name: plan-build
description: Break a requested feature into the smallest safe implementation plan before code changes.
disable-model-invocation: true
---

When invoked, do this:

1. Read the user request and current repo structure.
2. Identify existing pages, components, data structures, and likely touchpoints.
3. Propose the fastest implementation plan that preserves current UI quality.
4. Minimize rewrites.
5. Call out any likely breakpoints.
6. End with a numbered checklist of actions.

Do not start editing until the plan is written.
